Testing

It's not what it seems.

Because our first target is the authentication system, which is more or less straighforward in terms of complexity, we do not consider that Unit, Integration, Functional or User Acceptance testing are a must. Also, the nature of our system, the combination of mobile and web, makes the strategy even more difficult to tackle.

However, because we intend to follow the V model software development process, we will make sure that every part of our code is functional before we integrate it in the bigger repository.

We plan to use Loadster and Netsparker to test our web application for loading times and possible security vulnerabilities. Furthermore, PredictionIO's evaluation module allows to streamline the process of testing the engine parameters and deploy only what is acceptably accurate using statisically sound cross-validation methods.
