Development Plan

Following gathering all the requirements in Term 1, we created a Gantt Chart for the plan of development in Term 2 (this can be found on the Management section).

The plan for development was split into the stages, which can be seen below:

  • Implement the PredictionIO engine and train with data throughout the course of the term
  • Complete front-end of iOS app and the Touch ID implementation before other functionality is added.
  • Set up Django web app with working PostgreSQL user database, which will be the backend for both iOS and web apps.
  • Integrate the iOS app with the web app so that the iOS app can access the shared PostgreSQL database and implement the password login functionality for both apps.
  • Complete tuning and training of PredictionIO engine.
  • Integrate the PredictionIO engine and event server with the iOS app, so that prediction queries can be made and login data can be sent to the event server every time there is a successful login.
  • Deploy all server side components of the system onto Bluemix (i.e. PredictionIO engine, Django web app etc.).
  • Implement Push Notification Login system on the iOS and web app (using APNs and IBM Bluemix Push Notification Service).
  • Implement SMS Login System on the iOS and web app (using web sockets and an external SMS API).

Iterations

Iteration 1

iOS App

  • iOS app front end was implemented (i.e. 2 screens - phone number screen and password screen)

Web App

  • Django project was set up. Views created with plain HTML responses.
  • URL mapping implemented with no functionality.

    • Iteration 2

    iOS App

    • The iOS app was connected to the PostgreSQL user database via PHP GET requests for the iOS login authentication system, so that the user’s phone number is verified before loading the password screen and the password also being checked in a similar way (this was initially tested locally).
    • Touch ID (fingerprint) login was implemented at this stage, this consisted of asking the user for permission and implementing error checking for different contexts for when the Touch ID login failed (for example, the user cancels Touch ID login).

    Web App

  • Local database set up with nesesscary Django database models.
  • PostgreSQL database added to the Django settings file, database migrations made and applied.
  • Responsive forms added.
  • Passwords encrypted in the database.
  • REST API implemented for JSON data transfer.

    • Iteration 3

    iOS App

    • iOS login authentication system implementation was changed to use JSON data collected from hidden URL from the web app, instead of the less secure method of sending GET requests to verify the user’s phone number and password.
    • Welcome screen was added to the front end of the app to act as feedback for the user.

    Web App

  • Session functionality added in order to support queries with encrypted data between URLs.
  • Custom CSS and HTML templates added to create more appealing UI.

    • Iteration 4

    iOS App

    • PHP files were implemented for REST calls to query the prediction engine and sending events to the event server
    • The iOS app was connected to the prediction engine and able to send events to the event server and query for prediction results via PHP. The data required for these queries were sent to the PHP files via GET requests (time, day and location coordinates).
    • Permission options were also implemented into the iOS app (asking the user for permission to use Location Services and Prediction Login)
    • A settings bundle was added so that the user is able to enable/disable the different login options

    Web App

  • Custom fonts added, front-end polishing complete.
  • Django debug mode set to False in order to improve security.
  • Static files served through DJ-Static Django API.
