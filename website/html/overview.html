Final Requirements For Final PoC, Development Plan and Iterations

Final Requirements

We have compiled a list of final requirements for the Final Proof of Concept, chosen from the MoSCoW Requirements List and additional requirements .

Description Type Category Priority
E & S Banking system shall be accessible from any device that supports internet conection Non-Functional General Must Have
E & S Banking system shall use secure encryption methods Non-Functional Security Must Have
E & S Banking system shall adapt to it's users behaviour by using machine learning algorithms Non-Functional General Must Have
E & S Banking system shall have all parts of the system deployed onto IBM Bluemix Non-Functional Development Must Have
E & S Banking system shall have both web and native mobile apps Non-Functional General Must Have
Mobile App shall automatically provide instant access to users who satisfy the machine learning algorithm conditions Functional Authentication Must Have
Mobile App shall authenticate users via fingerprint scanner Functional Authentication Must Have
Machine learning algorithm shall collect data from both mobile login attempts and store them in a database Functional Development Must Have
Failsafe login method shall allow users to access their bank account by using default authentication procedure set by the bank(i.e. password login) Functional Authentication Must Have
Machine learning algorithm shall model user's behaviour by analysing geolocation and time Functional Development Must Have

Iterations

Iteration 1

iOS App

  • iOS app front end was implemented (i.e. 2 screens - phone number screen and password screen)

Web App

Iteration 2

iOS App

  • The iOS app was connected to the PostgreSQL user database via PHP GET requests for the iOS login authentication system, so that the user’s phone number is verified before loading the password screen and the password also being checked in a similar way (this was initially tested locally).
  • Touch ID (fingerprint) login was implemented at this stage, this consisted of asking the user for permission and implementing error checking for different contexts for when the Touch ID login failed (for example, the user cancels Touch ID login).

Web App

Iteration 3

iOS App

  • iOS login authentication system implementation was changed to use JSON data collected from hidden URL from the web app, instead of the less secure method of sending GET requests to verify the user’s phone number and password.
  • Welcome screen was added to the front end of the app to act as feedback for the user.

Web App

Iteration 4

iOS App

  • PHP files were implemented for REST calls to query the prediction engine and sending events to the event server
  • The iOS app was connected to the prediction engine and able to send events to the event server and query for prediction results via PHP. The data required for these queries were sent to the PHP files via GET requests (time, day and location coordinates).
  • Permission options were also implemented into the iOS app (asking the user for permission to use Location Services and Prediction Login)
  • A settings bundle was added so that the user is able to enable/disable the different login options
