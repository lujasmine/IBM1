Initial Requirements, MoSCoW Requirements and Use Cases

Initial Requirements Gathering

In our initial meeting with the client, we discussed the idea behind Effortless and Secure Banking and the possibilities of the project.

Following this, the client requested that in the second meeting, we would present our outlook on the project title and ideas for the solution.Below is the presentation that we presented:

MoSCoW Requirements List


Description Type Category Priority
E&S Banking system shall be accessible from any device that supports internet conection Non-Functional General Must Have
E&S Banking system shall use secure encryption methods Non-Functional Security Must Have
E&S Banking system shall adapt to it's users behaviour by using machine learning algorithms Non-Functional General Must Have
E&S Banking system shall have database backup Non-Functional General Must Have
E&S Banking system shall have both web and native mobile apps Non-Functional General Must Have
Mobile App shall automatically provide instant access to users who satisfy the machine learning algorithm conditions Functional Authentication Must Have
Mobile App shall authenticate users via fingerprint scanner Functional Authentication Must Have
Web Application shall send out a text message with authentication code during the login attempt Functional Authentication Must Have
Machine learning algorithm shall collect data from both mobile and web login attempts and store them in a database Functional Development Must Have
Failsafe login method shall allow users to access their bank account by using default authentication procedure set by the bank Functional Authentication Must Have
Machine learning algorithm shall model user's behaviour by analysing geolocation and time Functional Development Must Have
System for temporary transaction locks and limitations shall be implemented Functional Development Should Have
System shall display a map of transaction locations Functional Development Could Have
E&S Banking system shall be available in multiple languages Non-Functional Localization Could Have
Step-by-step introduction to the application Non-Functional User Experience Could Have
